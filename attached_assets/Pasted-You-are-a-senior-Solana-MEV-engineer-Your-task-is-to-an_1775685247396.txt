## ENDPOINT USAGE (STRICT)

### GET /order

* ONLY for **opportunity detection**
* Returns: quote, outAmount, routePlan, priceImpact
* Uses all routers → best price
* **Do NOT** use its transaction for MEV execution

### GET /build

* ONLY for **building atomic transactions**
* Returns: swap instructions, account metas, and Address Lookup Tables (ALTs)
* Metis routing only
* Mandatory for MEV: allows injection into atomic tx

### POST /execute

* **DO NOT** use in MEV system
* Managed execution → breaks atomic arbitrage
* Prevents custom instruction injection

## SYSTEM ARCHITECTURE

### detection.rs

* Calls GET /order
* Parses quote, routePlan, priceImpact
* Detects profitable opportunities

### swap_v2.rs

* Calls GET /build
* Extracts swap instructions, account metas, and ALTs
* Fetches ALTs from chain and constructs MessageAddressTableLookup entries
* Validates all accounts exist → **no mocks/subs allowed**

### flash_tx_v2.rs / atomic_tx.rs

* Builds **VersionedTransaction v0**
* Combines:

  * ComputeBudget
  * FlashLoanBorrow
  * Swap instructions with account metas
  * FlashLoanRepay
  * Optional Jito tip
* Ensures exact repayment amounts
* Discards any instruction referencing placeholder, dummy, or mock accounts

### signer.rs

* Signs with all required keypairs:

  * Payer
  * Flash loan authority (if required)
  * Any PDA signer via seeds

### executor.rs

* Simulates transaction **before sending**
* On simulation failure → DO NOT send
* On success → send via Jito bundle or RPC
